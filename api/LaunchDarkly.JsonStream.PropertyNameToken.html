Struct PropertyNameToken

Used by ObjectReader to return an object property name in situations where it may not be desirable to allocate a string instance.

Remarks

This type is used for the Name property of ObjectReader, to allow for a slight efficiency gain in some circumstances by not always having to create a string instance. The rationale is that JSON property names are often used only briefly for comparisons during parsing, and if they do not need to be retained in the parsed data, it is preferable to avoid creating many string instances that will increase garbage collection overhead.

Whenever possible, the parser will set the PropertyNameToken to work as a direct view onto the underlying character sequence in the JSON input, in which case using Equals(String) or == will not cause any allocations. In some cases (such as if the property name contains string escape sequences) that is not possible, and the parser will have to allocate a string, but will wrap it in a PropertyNameToken so your code can treat it the same either way.

The down side is that it is not possible to use a PropertyNameToken as the value in an ordinary switch statement; if you want to use switch, you must either call ToString() to get the name as a string, or use when clauses in the switch as shown in the example. But even though switch can be slightly faster than a series of equality tests, if the number of comparisons is small that advantage may not matter. 

    string prop1Value, prop2Value;
    for (var obj = reader.Object(); obj.Next();)
    {
        switch (obj.Name)
        {
            case var n when n == "prop1":
                prop1Value = obj.String();
                break;
            case var n when n == "prop2":
                prop2Value = obj.String();
                break;
        }
    }
