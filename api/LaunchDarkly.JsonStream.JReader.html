Remarks

On platforms that provide the System.Text.Json API (.NET Core 3.1+, .NET 5.0+), this works as a wrapper for System.Text.Json.Utf8JsonReader, providing a more convenient API for common JSON parsing operations. On platforms that do not have System.Text.Json, it falls back to a portable implementation that is not as fast as Utf8JsonReader but still highly efficient and has no external dependencies. Portable JSON unmarshaling logic can therefore be written against this API without needing to know the target platform.

The general usage pattern is as follows:

JReader is defined as a ref struct, which places certain limitations on how it can be used in .NET code. This is done for two reasons. First, this is necessary in order for it to be able to take advantage of System.Text.Json on compatible platforms. Second, it allows the compiler to make significant optimizations regardless of the platform, since it guarantees that this object will never be allocated on the heap. An indirect result of this design is that methods like Next(ref JReader) and Next(ref JReader) require the original JReader to be passed as a ref parameter, because they are not allowed to retain a reference to it in a field of their own.

Another design decision based on System.Text.Json interoperability and efficiency is that JReader methods can throw either their own exception types (derived from JsonReadException), or, on some platforms, a System.Text.Json.JsonException. JReader does not automatically catch the latter and translate them to its own types, in order to avoid the overhead of try blocks within high-traffic code paths. If you need to be able to distinguish between different kinds of JSON errors, you can call TranslateException(Exception) after catching an exception.

TranslateException(Exception)

Normalizes exceptions that may have been thrown from low-level code.

Declaration
public Exception TranslateException(Exception e)
Parameters
Type Name Description
System.Exception e

the exception that was thrown
Returns
Type Description
System.Exception

a SyntaxException or TypeException if the original exception was a different type that can be translated to those types; otherwise, the original exception
Remarks

When using a target framework that has System.Text.Json, some low-level JSON parsing logic may throw a System.Text.Json exception type rather than the JsonReadException types used by JReader. To avoid the overhead of using try/catch in high-traffic code paths, JReader does not translate these exceptions on a per-method-call basis. Instead, if you want to be able to distinguish between different kinds of errors in a consistent way, you should catch Exception in general and then pass the exception to this method.
