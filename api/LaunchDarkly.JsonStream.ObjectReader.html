Class ObjectReader

Used in conjunction with JReader to iterate through a JSON object.

Inheritance
System.Object
ObjectReader
Namespace: LaunchDarkly.JsonStream
Assembly: LaunchDarkly.JsonStream.dll
Syntax
public sealed class ObjectReader : ValueType
Remarks

The Object() and ObjectOrNull() methods return an instance of this type. Call the Next(ref JReader) method to iterate through each object property. Properties may appear in any order. To read the actual values, you will still use the JReader's methods.

This example reads an object whose values are strings; if there is a null instead of an object, it behaves the same as for an empty object. 

    var values = new Dictionary<string, string>()
    for (var obj = reader.ObjectOrNull(); obj.Next(ref reader);)
    {
        values[obj.Name.ToString()] = reader.String()
    }

The next example reads an object with two expected property names, "a" and "b". Any = unrecognized properties are ignored. 

    int aValue, bValue;
    for (var obj = reader.ObjectOrNull(); obj.Next(ref reader);)
    {
        switch (obj.Name)
        {
            case var n when n == "a":
                a = reader.Int();
                break;
            case var n when n == "b":
                b = reader.Int();
                break;
        }
    }

Note that this switch block uses when clauses rather than simple case "a":, etc., because obj.Name is not a string. For details, see PropertyNameToken. You could also do the following, which is simpler but may cause more heap allocations: 

    int aValue, bValue;
    for (var obj = reader.ObjectOrNull(); obj.Next(ref reader);)
    {
        switch (obj.Name.ToString())
        {
            case "a":
                a = reader.Int();
                break;
            case "b":
                b = reader.Int();
                break;
        }
    }

Remarks

It returns false if the JReader has reached the end of the object, or if the object was empty or null.

If Next returns true, you can then use Name to check the name of the property, and use JReader methods such as Bool() to read the element value. If you do not care about the value, simply calling Next again without calling a JReader method will discard the value.

Remarks

This method returns a new, modified ObjectReader. It should be called before the first time you call Next(ref JReader). For instance: 

    var requiredProps = new string[] { "key", "name" };
    for (var obj = reader.Object().WithRequiredProperties(requiredProps); obj.Next(ref reader);)
    {
        switch (obj.Name) { ... }
    }

When the end of the object is reached, if one of the required properties has not yet been seen, Next(ref JReader) will throw a RequiredPropertyException.

For efficiency, it is best to preallocate the list of property names globally rather than creating it inline.
